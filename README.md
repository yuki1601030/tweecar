# Tweecar ユーザーテスト用プロトタイプ

Tweecar は、「人間が車SNSを直接見るサービス」ではなく、**人間には見えない裏側の匿名車SNSを自車AIが読み取り、必要な情報だけをドライバーに伝えるサービス**を説明するための空想プロトタイプです。

## このプロトタイプの目的

- 明日のユーザーテストで、車同士の匿名ネットワークと自車AIの役割を一目で理解してもらう
- ドライバーが見るのは、音声アシスタントやスマホチャット形式の安全な要約だけであることを伝える
- 急ブレーキ、渋滞、路面異常、駐車場混雑、休憩スポット需要などの匿名データを、自車AIが関連性・危険度・プライバシーの観点で選別する体験を見せる

## 人間側UIと車SNS内部ネットワークの違い

- **人間側UI**: ドライバーが利用する車内ディスプレイ風UIとスマホチャット風UIです。質問候補や音声入力風ボタンから、自車AIの自然文回答だけを確認できます。
- **自車AI中間レイヤー**: 見えない車SNSのデータを、関連性判定、危険度判定、複数車両での裏取り、プライバシー保護、要約・最適化の順に処理する説明用レイヤーです。
- **車SNS内部ネットワーク**: 実際のユーザー画面ではなく、説明のために表示している裏側イメージです。車同士が匿名データ信号を共有している様子を示しますが、生の投稿本文や他車の会話は表示しません。

## GitHub Pagesでの確認方法

このプロトタイプは `index.html` 1ファイルで完結しています。外部API、ログイン、DB、ビルド工程は不要です。

1. リポジトリを GitHub に push します。
2. GitHub の **Settings → Pages** を開きます。
3. Source を対象ブランチ、Folder を `/ (root)` に設定します。
4. 発行された GitHub Pages URL を開くと、そのまま `index.html` が表示されます。

ローカルで確認する場合は、ブラウザで `index.html` を直接開くか、任意の静的サーバーで表示してください。

## データとプライバシーについて

これはユーザーテスト用の空想プロトタイプです。実際の車両データ、個人情報、車両ID、ナンバー、正確な位置情報、実在ユーザーの投稿本文は扱っていません。画面内の情報はすべてダミーデータです。
