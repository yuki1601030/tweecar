# TweeCar プロトタイプ

TweeCar は、**街中の車たちが裏側で匿名情報を共有し、自車AIがドライバーに必要な情報だけ届ける**サービスのユーザーテスト用静的プロトタイプです。

## このプロトタイプの目的

- 「車たちが裏で会話して、自分の車が必要情報だけ教えてくれる」体験を一目で理解してもらう
- 人間が車SNSの投稿を読むサービスではなく、AIが匿名情報を要約して届けるサービスであることを伝える
- 渋滞、工事、駐車場混雑、急ブレーキ、路面状況、休憩スポットなどのダミーデータで利用イメージを確認する

## 2画面の違い

- **サービスイメージ画面**  
  明るい街のビジュアル上に、車同士の匿名情報共有を概念図として表示します。吹き出しは世界観説明用で、実際のドライバー画面ではありません。

- **ユーザー体験画面**  
  実際にドライバーが使う想定の画面です。左に車内ディスプレイ風の音声アシスタント、右にスマホチャット連携UIを配置しています。

## ドライバーに見えないもの

- ドライバーは他車の投稿本文を直接見ません
- 他車の会話を読むことはできません
- 個人情報、車両ID、ナンバー、正確な位置情報は表示しません
- AIが自車に関係ある情報だけを選別・要約して届けます

## 入力体験

音声入力中心の想定です。マイクボタンを押すと、以下の流れを演出します。

1. 聞き取り中...
2. 車SNSから関連情報を確認中...
3. 回答生成中...
4. AI回答を表示

同じ内容はスマホチャットにも連動する想定で、質問候補ボタンから会話を追加できます。

## GitHub Pagesでの確認方法

このプロトタイプは `index.html` 1ファイルで完結しています。外部API、ログイン、DB、ビルド工程は不要です。

1. リポジトリを GitHub に push します。
2. GitHub の **Settings → Pages** を開きます。
3. Source を対象ブランチ、Folder を `/ (root)` に設定します。
4. 発行された GitHub Pages URL を開くと、そのまま `index.html` が表示されます。

ローカルでは `index.html` をブラウザで直接開くか、任意の静的サーバーで確認できます。
